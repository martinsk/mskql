Notes

Implementation details, measured.

Each note describes one optimization: the query used to measure it, what the code was doing before, what changed, and the before/after numbers. All measurements use the same benchmark harness (bench-vs-pg) on the same machine.

  1. Result cache and join cache
    Wire-byte result cache + hash-join build-side cache. mixed_analytical throughput: 97,210 QPS vs PostgreSQL 508 QPS (191×).
  2. Disk-backed tables: near-zero overhead persistence
    CREATE DISK TABLE adds persistence at 0.96× the in-memory speed.
  3. Vectorized expression aggregates (VEC_PROJECT)
    Direct column-array arithmetic replaces per-row expression evaluation. expression_agg: 15ms vs PostgreSQL 299ms (20×).
  4. Top-N sort: fused SORT + LIMIT via binary heap
    Binary max-heap of size K avoids full sort. subquery_complex with LIMIT 500 on 50K rows.
  5. Correlated subquery decorrelation
    Materialized aggregate + LEFT HASH_JOIN replaces O(N×M) nested loop. 4ms vs PostgreSQL 3,636ms (826×).
  6. CTE materialization: from legacy fallback to plan executor
    Pre-materialize non-recursive CTEs as temp tables before plan build. mixed_analytical CTE pipeline fully in the columnar executor.
  7. Radix sort for integer ORDER BY
    8-bit radix sort replaces comparison-based sort for integer keys. large_sort: 77ms vs PostgreSQL 209ms.
  8. Zero steady-state allocation: the arena model
    Bump-slab arena with pointer rewind replaces malloc/free per query. aggregate: 15ms vs PostgreSQL 265ms (18×).
  9. Closing the ORDER BY gap: 4.65× slower to 1.18×
    Direct columnar→wire serialization + flat sort comparator. order_by: 1,676ms → 429ms.
